# Web Server 演化主线

## 目标
通过一个不断扩展的 Web Server，理解后端系统为什么会逐步引入更多机制。

## 计划中的阶段
1. `stage-00`：最小可用的单进程 Web Server
2. `stage-01`：请求并发带来的处理模型变化
3. `stage-02`：使用线程池或工作池控制资源成本
4. `stage-03`：引入数据库后的新瓶颈
5. `stage-04`：数据库压力上升后的优化路径
6. `stage-05`：引入缓存后的收益与代价
7. `stage-06`：使用队列处理异步任务
8. `stage-07`：单机不够时引入多实例和负载均衡
9. `stage-08`：加入可观测性与稳定性设计

## 使用方式
主线负责说明“为什么走到这一步”。
如果某一步涉及独立知识点，就链接到 `docs/topics/`。
如果某一步需要观察行为，就链接到 `labs/` 中对应的阶段目录。
