# 知识点文档

这里记录可以从主线跳转过来的具体知识点。

## 这一类文档的职责
每篇知识点文档建议优先回答：
- 这个技术或概念是什么
- 它解决什么问题
- 在工程里有哪些主要用法
- 什么时候适合使用
- 常见坑和注意事项是什么
- 需要去哪里看对应代码实验

## 当前阶段的组织原则
当前项目优先按“一个主题一篇主文档 + 一个独立实验项目”的方式组织。

例如并发主题更推荐：
- 一篇主文档统一说明武器、用法、适用场景和常见坑
- 一个独立实验项目统一做代码演示和横向比较

这样做的原因是：
- 更贴近人类学习时的阅读路径
- 减少在多个文档之间来回跳转的成本
- 更符合当前项目强调全景理解的目标

## 这一类文档不负责什么
- 不负责讲完整系统如何一步步演化
- 不负责承担主线叙事
- 不负责替代 runnable demo

## 和其他目录的关系
- 它通常由 `docs/roadmap/` 中的某一步链接过来
- 如果需要观察行为，应继续链接到 `labs/`
