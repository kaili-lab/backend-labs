# 能力地图目录

这里记录后端工程能力的高层地图，用来承接“核心原则”和“专题细节”之间的空档。

## 这一类文档的职责
能力地图文档负责回答：
- 一个后端工程师实际要具备哪些能力块
- 核心原则如何映射到工程工作
- 每类能力为什么会出现、为什么重要
- 每类能力通常会延伸到哪些专题文档和实验
- 学习时应该先看什么、再深入什么

它的重点是建立“原理 -> 能力域 -> 专题”的过渡层。
所以这一类文档更像“城市地图”——先告诉你大区怎么分，再决定往哪条街走。

## 这一类文档不负责什么
- 不负责展开所有实现细节
- 不负责替代专题知识文档
- 不负责展示完整代码实验
- 不负责把每个技术点讲成操作手册

## 和其他目录的关系
- 如果你在看“后端为什么会复杂、底层矛盾是什么” -> 回到 `docs/java-topics/后端开发核心原则.md`
- 如果你在看“一个后端工程师要掌握哪些能力域” -> 看当前目录
- 如果你在看“某个专题内部怎么展开” -> 去 `docs/java-topics/`
- 如果你在看“系统为什么在某个阶段演化” -> 去 `docs/roadmap/`
- 如果你在看“代码跑起来会怎样” -> 去 `labs/`

## 当前包含的文档
- `docs/capability-map/后端工程能力地图.md`：回答“后端工程师要具备哪些能力域”
- `docs/capability-map/后端演进线.md`：回答“这些技术通常是怎样一步一步被逼出来的”，并给出小业务例子与映射关系
