# backend-labs

一个以后端系统演化为主线的学习型项目，用来连接概念、场景、取舍和代码实验。

## 为什么做这个项目

这个项目的目标不是堆积知识点，而是建立一套更清晰的后端认知地图。
重点不是单独理解某个框架或技术，而是理解：
- 为什么需要它
- 什么时候该用它
- 它解决的核心问题是什么
- 面对类似方案时应该怎么选择

## 学习方式

- 从一个很小的系统开始
- 随着压力增加逐步演化系统
- 先分析问题，再引入技术
- 以项目演进为主线
- 以自己的专题整理作为知识沉淀方式
- 允许边做边调整，不追求一开始就设计完美

## 各部分作用

- `项目记忆.md`：记录项目长期记忆，包括目标、结构原则、当前决定和路线
- `领域知识整理方法论.md`：记录可迁移到其他领域的通用知识整理框架
- `学习路径.md`：记录建议学习顺序
- `docs/ai-application/`：记录 AI 应用开发方向的初始判断和后续整理入口
- `docs/roadmap/`：记录系统从简单到复杂的演化主线
- `docs/capability-map/`：记录后端工程能力的高层地图
- `docs/java-topics/`：记录你围绕专题整理的知识文档
- `labs/web-server-scaling/`：主实验项目，模块化单体商城，配合演进线逐步迭代
- `labs/independent-demos/`：独立的小型实验，用来观察某个局部机制
- `references/`：你过去整理的知识体系与检查清单，作为后续专题整理时的参考材料

## 四类内容的职责边界

### 1. 主线文档：`docs/roadmap/`
主线文档负责回答：
- 系统现在遇到了什么问题
- 为什么这个问题会在当前阶段出现
- 应该从哪些角度分析
- 有哪些可选方案
- 为什么当前先选择这个方案
- 引入方案后会增加什么复杂度

它的重点是**串联演化过程**，不是展开解释所有技术细节。
所以主线文档更像“剧情线”或“路线图”。

### 2. 能力地图：`docs/capability-map/`
能力地图文档负责回答：
- 一个后端工程师实际要具备哪些能力块
- 核心原则如何映射到工程工作
- 每类能力为什么会出现、为什么重要
- 每类能力通常会延伸到哪些专题和实验
- 学习时应该先看全局还是先钻细节

它的重点是**建立“原理 -> 能力域 -> 专题”的过渡层**，不是立即展开全部实现细节。
所以能力地图更像“城市地图”——先告诉你区域怎么分，再决定往哪条街走。

### 3. 专题文档：`docs/java-topics/`
专题文档负责回答：
- 某个专题内部有哪些核心知识点
- 当前 stage 用到了这个专题中的哪一部分
- 各个知识点分别解决什么问题
- 这些知识点之间是什么关系
- 需要特别注意哪些坑和边界

它的重点是**由你自己整理和沉淀理解**，而不是只做主线的附属注释。
所以专题文档更像“你自己的知识图谱”。

### 4. 实验项目：`labs/`
实验项目负责回答：
- 这个方案跑起来是什么样
- 能观察到什么现象
- 不同实现方式会带来什么行为差异
- 当前方案在代码层面是怎么落地的

它的重点是**观察行为和验证想法**，不是替代主线文档或专题文档。
所以实验项目更像“实验室”。

## 怎么判断内容该放哪里

- 如果你在讲“系统为什么演化到这一步” -> 放 `docs/roadmap/`
- 如果你在讲“一个后端工程师要掌握哪些能力域” -> 放 `docs/capability-map/`
- 如果你在整理“某个专题本身的知识结构” -> 放 `docs/java-topics/`
- 如果你在讲“代码跑起来会发生什么” -> 放 `labs/`
- 如果你在回看过去已有的知识提纲或检查清单 -> 看 `references/`

## 当前建议

如果你想理解这个项目背后的通用整理方法，先读 `领域知识整理方法论.md`。
如果你在建立后端全景认知，建议再读 `docs/java-topics/后端开发核心原则.md` 和 `docs/capability-map/后端工程能力地图.md`。
如果你准备开始整理 AI 应用开发方向，可以先读 `docs/ai-application/AI应用开发方向判断.md`、`docs/ai-application/AI应用开发核心原则.md`、`docs/ai-application/AI应用开发能力地图.md` 和 `docs/ai-application/AI应用开发演进线.md`。
然后回到 `docs/roadmap/` 推进主线，再围绕当前演进阶段真实涉及的专题，到 `docs/java-topics/` 下整理知识结构与思考。
