# 学习路径

## 第一阶段：先建立整体地图
- 理解项目目标和学习方式
- 学会用“压力、瓶颈、取舍”来思考后端问题
- 建立一个最基础的请求处理认知模型

## 第二阶段：从最小系统开始
- 实现一个最简单的 Web Server
- 理解请求从进入到响应的完整过程
- 观察最基础的延迟、吞吐和资源占用

## 第三阶段：随着压力逐步扩展
- 当串行处理成为瓶颈时，引入并发模型
- 当资源创建成本过高时，引入池化
- 当单进程或单机压力过大时，拆分职责
- 当瓶颈明确出现时，再引入缓存、队列、负载均衡等方案

## 第四阶段：补足工程能力
- 加入可观测性
- 加入稳定性与弹性设计
- 理解一致性、故障模式和运行时权衡

## 第五阶段：回头比较方案
- 回看之前的关键决策
- 比较不同方案在不同约束下的适用性
- 逐步形成自己的后端决策框架
