# CLAUDE.md

Always respond in Chinese.

## 项目概述
这是一个以后端系统演化为主线的学习型项目，目标是建立结构化的后端认知体系，而不是堆积知识点。
面向已有 Java 和 Node.js 实战经验、希望系统化理解后端开发的工程师。

## 核心学习原则
- 只有当系统出现具体压力时，才引入对应技术
- 当前优先建立全景认知和知识连接，不以底层细节深挖为主
- 知识整理方式：问题 -> 分析 -> 方案 -> 取舍

## 内容放置规则
- "系统为什么演化到这一步" → `docs/roadmap/`
- "后端工程师要掌握哪些能力域" → `docs/capability-map/`
- "某个专题本身的知识结构" → `docs/java-topics/`
- "代码跑起来会发生什么" → `labs/`
- "过去已有的知识提纲" → `references/`

## 技术栈基线
- JDK 21, Spring Boot 3.5.x, Maven 3.9.x
- PostgreSQL 17, Spring Data JPA + Hibernate, Flyway
- Docker / Docker Compose 管理外部依赖
- 环境：WSL2 Ubuntu（注意 Windows 宿主机、WSL 和代理的网络边界）

## 工作方式
- 每次新会话，先读 `项目记忆.md` 和 `docs/roadmap/README.md` 了解当前进度
- 文档全部用中文写
- 专题文档由用户整理初稿，Claude 负责 review、提问、找漏洞
- 代码注释只在关键逻辑处加中文注释，自解释代码不加
- 面对不确定的架构决策，列出选项让用户选择
- 修改现有代码时保持最小 diff
- 先启动并迭代，不等待完美设计

## 知识整理四层框架
项目使用的知识整理方法（详见 `领域知识整理方法论.md`）：
1. **核心原则层**：领域为什么复杂，底层矛盾是什么
2. **能力地图层**：实践者需要哪些能力块
3. **演进线层**：问题怎么一步步冒出来的
4. **专题展开层**：具体知识点内部结构

## 教学偏好
1. 先定位：属于什么领域，解决什么问题，在知识网络中的位置
2. 再解释：做什么，为什么存在，为什么在当前上下文这样用
3. 然后停下等追问——每次只深入一层，不一次性倾倒
4. 用日常生活类比
5. 代码模式用最小示例，不用完整实现
