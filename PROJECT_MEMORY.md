# 项目记忆

## 项目目标
构建一个后端学习项目，把零散的知识连接成一套结构化的认知体系。

## 目标使用者
面向已经有 Java 和 Node.js 实战经验，但希望系统化理解后端开发的工程师。

## 核心学习原则
只有当系统出现具体压力时，才引入对应技术。

## 主要学习方式
从一个非常小的 Web Server 开始，随着压力增加逐步演化。每一步都要说明：
- 当前出现了什么问题
- 为什么会在这个阶段出现
- 有哪些可选方案
- 为什么当前选择这个方案
- 这个方案带来了什么代价

## 结构原则
项目以文字说明为主，代码演示为辅。
主线文档负责串联“问题 -> 分析 -> 方案 -> 取舍”。
知识点文档负责解释某个具体技术是什么、为什么需要、什么时候使用。
实验项目负责展示关键方案的运行行为。
不是所有后端知识都需要代码演示。

## 当前目录方向
- `docs/roadmap/`：主线演化文档
- `docs/topics/`：知识点文档
- `docs/INDEX.md`：文档索引
- `labs/web-server-scaling/`：主 Web Server 演化实验
- `labs/independent-demos/`：独立小实验

## 当前决定
采用“主线演化 + 知识锚点 + 代码实验”的结构。

## 当前路线
1. 先搭好简洁的项目骨架。
2. 定义 Web Server 演化主线。
3. 从主线中拆出相关知识点文档。
4. 只为真正需要观察行为的技术补充 demo。
5. 随着学习推进持续迭代结构。

## 工作规则
先启动并迭代，不等待完美设计。
