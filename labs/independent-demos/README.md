# 独立实验

这里放与主线相关但不适合直接放入主实验中的小型 demo。

## 这个目录的职责
- 用来验证某个局部机制的行为
- 用来做对比实验或现象观察
- 用来补充主线实验中不适合展开的小主题

## 组织原则
独立实验优先按“主题”聚合，而不是按单个 API 或单个技巧零碎拆分。

例如更推荐：
- `java-concurrency-lab/`
- `database-access-lab/`
- `caching-lab/`
- `async-messaging-lab/`

而不是一开始就拆成很多零散的小项目。

这样设计的原因是：
- 更方便比较同一类方案的差异
- 更容易形成完整的武器库视图
- 更适合当前项目强调系统化理解的目标

## 适合放在这里的内容
- 并发相关主题实验
- 连接池行为实验
- 缓存相关现象实验
- 队列重试或积压实验

## 不适合放在这里的内容
- 主线阶段的正式实现
- 没有明确学习目标的临时代码
- 需要由主线来解释背景的问题
