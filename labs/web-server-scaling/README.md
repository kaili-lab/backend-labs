# Web Server 主实验

这是整个项目的主实验，用来配合主线文档逐步演示系统演化。

## 这个目录的职责
- 承接 `docs/roadmap/服务扩展演进路线.md` 中的主线阶段
- 用最小可运行代码展示当前阶段的实现方式
- 帮助观察某个方案的运行现象和局限

## 这个目录不负责什么
- 不负责解释完整的技术背景
- 不负责替代主线中的问题分析和方案取舍
- 不负责成为杂乱的 demo 收纳箱

## 目录约定
每个阶段目录都应包含：
- `README.md`：说明当前阶段的问题、方案和取舍
- 可运行代码：用于演示当前阶段的最小实现
- 运行方式：告诉使用者如何启动
- 观察说明：告诉使用者应该看什么现象
