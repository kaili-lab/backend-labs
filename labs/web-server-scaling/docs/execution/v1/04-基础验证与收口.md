# V1-04：基础验证与收口

## 本批目标

对 `V1-01 ~ V1-03` 的成果做最小验证、补齐运行说明，并把 V1 收口成一个真正可运行的最小版本。

本批完成后，应达到：

- V1 范围内的 happy path 可以验证
- 最小 smoke test 存在
- 基础运行说明完整

## 本批范围

- 补最小 smoke test
- 校验关键状态流转：
  - 商品读取
  - 创建订单
  - 确认支付
  - 通知任务执行
- 补最小运行说明
- 补基础环境配置说明
- 回看 V1 文档与实际实现是否一致

## 本批明确不做

- 大规模测试体系
- 压测
- V2 的幂等与可靠性增强
- Redis 缓存优化

## 验收标准

### 自动化验收

- 至少存在一条最小 happy path 的 smoke test，覆盖：
  - 商品读取
  - 创建订单
  - 确认支付
  - 通知任务结果观察

### 手动验收

- 存在一份可复现的最小运行说明
- 按说明可以完成从启动应用到走通 happy path 的验证

### 一致性验收

- 当前实现与以下文档的主口径保持一致：
  - `docs/slices/V1-跑通核心交易链路.md`
  - `docs/designs/v1/01-最小业务模型设计.md`
  - `docs/designs/v1/02-核心交易链路与事务边界设计.md`

### 收口验收

- 可以明确判断 V1 已具备最小可运行业务 happy path
- 超出 V1 边界的问题被记录为后续切片问题，而不是继续顺手扩展

## 风险与注意点

- 本批目标是“收口”和“验证”，不是继续扩功能
- 如果发现缺失项，只补 V1 范围内必须补的部分
- 超出 V1 边界的问题，应记录到 V2，而不是顺手扩展
