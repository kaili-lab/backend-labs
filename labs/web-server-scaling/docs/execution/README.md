# 执行清单与批次记录

这里存放真正开始编码前的执行清单，以及每一批实现完成后的记录。

## 这一层回答什么

- 这次具体改哪些文件
- 这次先做哪几步
- 哪些项已完成，哪些项还没做
- 是否有小范围调整，以及为什么调整

## 当前目录组织

- `v1/`：V1 相关执行清单与批次记录
- `v2/`：V2 相关执行清单与批次记录
- `v3/`：V3 相关执行清单与批次记录
- `v4/`：V4 相关执行清单与批次记录

## 使用约定

- 每次正式实现前，先在对应目录写一份简短执行清单
- 文件名优先使用 `01-主题.md`、`02-主题.md` 这种顺序
- 执行完成后，在同一文件或同目录补充完成情况
- 如果只是很小的改动，不必额外拆很多文档
