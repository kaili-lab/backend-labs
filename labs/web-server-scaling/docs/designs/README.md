# 关键设计文档

这里存放每个切片里值得单独展开说明的关键方案设计。

## 什么时候需要写设计文档

满足下面任一条件时，建议补设计文档：

- 有两个以上可选方案，需要明确取舍
- 会影响事务边界、状态流转或模块边界
- 会引入新的复杂度，需要提前讲清代价
- 用户后续很可能在面试或复盘里反复讲到这一点

不满足这些条件时，优先直接进入执行清单，不要把每个小功能都写成文档。

## 推荐写法

每份设计文档尽量包含：

- 背景与当前压力
- 候选方案
- 当前选择
- 为什么这样选
- 代价与边界
- 对后续切片的影响
